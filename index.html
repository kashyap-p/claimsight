The multi-modal, multi-agent claims adjudication copilot.

Ingest a messy insurance claim — scanned PDF, phone photo, recorded call, policy table — and a 9-agent workflow triages it, assesses damage from imagery, verifies coverage against the policy via RAG, scores fraud, and drafts a cited settlement recommendation. Shipped with a CI-gated eval harness and Langfuse-style tracing.

The problem it solves

After a single hurricane, a regional insurer can receive 50,000+ claims in 72 hours.

ℹ️
This is a portfolio project. The page you're reading is a static landing page hosted on GitHub Pages. The interactive app (with live agent streaming) needs a Node.js server — see how to run it locally or deploy to Vercel for a live demo.

✅ Works out of the box — clone and run, no API keys needed. The app runs in demo mode with realistic simulated agent responses. Configure .z-ai-config for real LLM calls (see README).
📋

Manual triage is slow

A human adjuster spends 30–45 minutes per claim reading documents, cross-referencing policies, checking for fraud, and writing a decision memo.

⏰

Catastrophic backlogs

50,000 claims in 72 hours means delayed payouts, customer frustration, and regulatory penalties for prompt-payment violations.

💰

Fraud is expensive

Insurance fraud costs the US P&C industry an estimated ~$45B/year. Manual review catches only a fraction.

🤖

ClaimSight automates this

9 AI agents work in parallel — triage, extraction, vision, RAG, fraud, coverage, adjudication, and QA — reducing triage from 45 min to ~6 min.

How it works — the 9-agent pipeline

Each agent is a real LLM call with a specialized prompt. Agents run with parallel fan-out and write to a typed shared state.

1 · Ingest
📦 Claim Bundle (PDF + photo + audio + tables)
↓
2 · Route
🧭 Supervisor → Intake Router (zero-shot classify)
↓ parallel fan-out
3 · Parallel Workers
📄 Doc Extractor (NER + Doc QA)
🔍 Policy Retriever (RAG)
👁️ Vision Assessor (Det + Seg + Classif)
↓ converge
4 · Assess
🚨 Fraud Detector (text + tabular)
✅ Coverage Verifier (clause match)
↓
5 · Adjudicate
⚖️ Adjudicator — decision + payout + cited clauses
↓
6 · QA Gate
🛡️ Auditor — groundedness + compliance → pass / retry / escalate
↓ human-in-the-loop
7 · Output
📄 Settlement memo + citations → adjuster review → payout

Run it yourself

The interactive app needs a Node.js server. Clone and run locally, or deploy to Vercel.

▲ Deploy to Vercel (free, one-click) 📦 View on GitHub
Node.js + npm
Bun (faster)
# 1. Clone the repo $ git clone https://github.com/kashyap-p/claimsight.git $ cd claimsight # 2. Install dependencies $ npm install # 3. (Optional) Copy env file $ cp .env.example .env # 4. Start the dev server $ npm run dev # → Open http://localhost:3000
💡 Then what?

Open the app → scroll to Claims Queue → click any claim (first has a START HERE badge) → click Run Pipeline → watch 9 agents execute live.
